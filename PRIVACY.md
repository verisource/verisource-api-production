# Privacy Policy - VeriSource Media Verification API

**Last Updated:** October 30, 2025

## Data We Collect

### Uploaded Files
- **Retention:** Files are processed in memory and deleted immediately after processing
- **Storage:** Temporary files exist for < 5 seconds during processing
- **Access:** Only accessible by the processing server, never stored permanently
- **Backup:** NO backups are made of uploaded files

### Fingerprints/Hashes
- **What we return:** Cryptographic hashes and fingerprints of your media
- **Storage:** We DO NOT store the fingerprints you generate
- **Ownership:** You own all fingerprints generated from your files

### Request Metadata
We temporarily log:
- IP addresses (for rate limiting and abuse prevention - 24 hours)
- Timestamps (for analytics - 7 days)
- File sizes and types (for analytics - 7 days)
- Processing times (for performance monitoring - 7 days)

We DO NOT log or store:
- File contents
- File names (beyond processing)
- User identities
- Fingerprint values

## GDPR Compliance

### Your Rights
- **Right to Access:** Request what data we have (none, except 7-day logs)
- **Right to Deletion:** Request deletion of your IP from logs
- **Right to Portability:** Download your data (if any)
- **Right to Object:** Object to processing (we don't store files anyway)

### Data Processing
- **Location:** Railway infrastructure (USA)
- **Purpose:** Media verification service only
- **Legal Basis:** Legitimate interest (service provision)

### Contact for GDPR Requests
Email: privacy@verisource.com

## Security Measures

- ✅ TLS/HTTPS encryption in transit
- ✅ No persistent file storage
- ✅ Rate limiting to prevent abuse
- ✅ File validation to prevent malicious uploads
- ✅ Regular security audits
- ✅ Automated file cleanup

## Third-Party Services

- **Railway:** Hosting provider (privacy policy: railway.app/legal/privacy)
- **FFmpeg:** Open-source media processing (no data sharing)

We DO NOT use:
- Analytics services (no Google Analytics, etc.)
- Advertising networks
- Third-party cookies
- CDNs that track users

## Data Breach Protocol

In the unlikely event of a data breach:
1. We will notify affected users within 72 hours
2. We will notify relevant data protection authorities
3. We will publish a public incident report
